Cracking passwords

Here you can find a cheatsheet for cracking some passwords. There will be uncommon things hard to remember...

John the reaper

    Specifying hash formats

      john hash.txt --format=afs

Hashcat

    Hashcat rules

      In case you found that you are not able to crack certain password with default wordlist you can use rules which provide variations of password presents in wordlists

      hashcat -m 0 hash.txt /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule

    Hascat masks

      In case you know that password contains certain format like (4 digits or first uppercase and other ones no...) we can specified in hashcat by using -a flag

      The following command means first uppercase (u) next four are lowercase (l) then a digit (d) finally a symbol (s) 

      hashcat -a 3 -m 0 1e293d6912d074c0fd15844d803400dd '?u?l?l?l?l?d?s'

Cewl

    In case you would like to find passwords based on the information of a company you could use cewl

    The following command is an example of how to scan a web for credentials (-d) depth of spider, (-m) min length of words

    cewl https://www.inlanefreight.com -d 4 -m 6 --lowercase -w inlane.wordlist

    Creating custom wordlist

      1. Situate the key words in a txt file

      2. Duplicate the key words by using

      hashcat --stdout \
-a 1 \
wordlist.txt \
wordlist.txt \
> new_wordlist.txt

      3. (Optional) remove all password less than 12 chars

      4. Create a custom rule, chars used can be

      :
c
so0
c so0
sa@
c sa@
c sa@ so0
$!
$! c
$! so0
$! sa@
$! c so0
$! c sa@
$! so0 sa@
$! c so0 sa@

      5. Apply custom rule

      hashcat --stdout \                                                                   
  -r custom.rule \
  new_second_wordlist.txt \ 
> final_wordlist.txt

      6. Finally use the final_wordlist.txt to crack the hash
