DNN - Dot Net Nuke (TCP/80)

CMS written in .NET, basically the WordPress of .NET

Default username

    Username: admin
Password: dnnadmin

Username: host
Password: dnnhost

    Password is changed during the installation so you aren't likely to get in by using that

Exploitation

    Once you are in there are a couple of entry vectors

    SQL console

      Accesing to the settings section (at the left part) you and then selecting SQL console you will enter in a SQL console which let's you type MSSQL commands

      This can be used to perform command execution into the server. For more information on how to do so check the MSSQL section in this website

      Check for more information ↗️

    Uploading a malicious webshell

      DNN also allow us to change the allowable file extensions to allow .asp and .aspx files or others. This can be done from the following section

      Security -> More -> More Security Settings and then adding them under Allowable File Extensions then click save button

      Then we can go the file management section

      http://172.16.8.20/admin/file-management

      Click in upload files button and select the ASP web shell we previously uploaded. Then right click on the uploaded file and select Get URL and you will have the command injection
