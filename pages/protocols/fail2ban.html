Fail2ban

Fail2ban is an open-source intrusion prevention framework for Linux that protects servers from brute-force and automated attacks

Privilege escalation

    In case we found that we have sudo privileges over a fail2ban binary we could be able to escalate our privilege. There are many things to do so even though we will focus on sudo privileges over the following command 

    fail2ban restart

    Fail2ban does have several configuration files located at /etc/fail2ban/ the main faile is jail.conf which contains all the inforation of what to to when an IP is banned, how many login attempts are permitted...

    If we look at it we could look at the following lines

    banaction = iptables-multiport
banaction_allports = iptables-allports

    Here we can see that whenever a banaction is triggered the action defined in iptables-multiport file will be executed. That file is located at /etc/fail2ban/action.d

    Since we are members of the group security we could write all files located at that folder (action.d) included the iptables-multiport. Now open that file and look for the following part

    #Tags: See jail.conf(5) man page
#Values: CMD
#
actionban = PUT YOUR MALICIOUS FILE PATH OR COMMAND HERE

    As we can see we are able to modify the action ban so once a user get's banned the code placed there will be executed. That way we could introduce the SUID into the /bin/bash with the following command

    chmod +s /bin/bash

    And then escalte with 

    /bin/bash -p
