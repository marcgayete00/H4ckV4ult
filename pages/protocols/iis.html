IIS (TCP/80 - TCP/443)

Internet Information Services, is web server developed by microsoft which is used to host wesites or web applications in windows environments

Enumeration

    Common paths for IIS

    C:\inetpub\
 C:\inetpub\wwwroot\

    Not confuse with intsrv (c:\windows\system32\inetsrv) where the configuration files are located

    IIS Tilde Enumeration

      IIS tilde directory enumeration is a technique utilised to uncover hidden files, directories, and short file names (aka the 8.3 format). The explanation of this format can be found here

      We can identify files based using the follwing script

      java -jar iis_shortname_scanner.jar 0 5 http://10.129.204.231/

      This script will be bruteforcing the filename using a,b,c,d... and so on. This way in case it finds a coincidence it will try again using aa,ab,ac,ad... The script outout shows

      |_ Number of sent requests: 553
|_ Identified directories: 2
   |_ ASPNET~1
    |_ UPLOAD~1
  |_ Identified files: 3
    |_ CSASPX~1.CS
      |_ Actual extension = .CS
    |_ CSASPX~1.CS??
    |_ TRANSF~1.ASP

      As we can see the script identifyed three files and two directories but it does not give us the all the filename. We can try by just simply using TRANSF~1.ASP but in case it does not work we'll have to create a wordlist. To do so we can use the following command

      This command will go over all files located at /usr/share/wordlist and will only select the words that start with transf creating a wordlist located at /tmp

      egrep -r ^transf /usr/share/wordlists/* | sed 's/^[^:]*://' > /tmp/list.txt

      Now use this wordlist combinaed with dirb or feroxbuster to reveal the file if the name matches
